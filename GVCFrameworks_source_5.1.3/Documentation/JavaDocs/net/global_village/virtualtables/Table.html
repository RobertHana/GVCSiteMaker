_VirtualTable
public class Table
extends _Table

The root object for a virtual table, which is a set of objects that can be created easily at runtime, that, together, act as if they were a single object (see VirtualRow) with a persistent store.

Table is the only object that you actually have to insert into the editing context. Columns and Rows are created by using newColumn() and newRow() respectively, and these methods automatically insert the newly created column or row into the table's editing context.

Usage is simple: choose the correct subclass of Table (only one to choose from right now, so that is simple ;-) and instantiate it and insert it into an editing context (there is a convenience method on VirtualTable to help you with this). Then use newColumn() to create columns on the new table and newRow to make new rows, and finally, use valueForFieldNamed() and takeValueForFieldNamed() or KVC to populate row data. For example: testTable = VirtualTable.createVirtualTable(editingContext(), "Test Table"); numberColumn = testTable.newColumn("number column", ColumnType.NumberColumnType); row = testTable.newRow(); row.takeValueForKey(new Integer(1), "number column");

Parameters:

sender - an arbitrary object that is used as a parameter to Column's valueForImportedValue() method
columnsToImport - an ordered list of columns that we will import. The first item in this array cooresponds to the first field in the data file, the second item to the second field, and so on
formatters - mapping of column names to java.text.Formats, used to format the field strings into objects of the correct type. You'll normally only need this for date/times and possibly for custom column types
delimiter - the string to use as the delimiter
skipFirstLine - should we skip the first line? Useful if, for example, the first line contains the column names
shouldReplace - true if this should replace all the rows, false if it should append the rows
Throws:
IOException - occurs when an IO exception occurs on the steam
ParseException - occurs when a formatter cannot format the data given it
