public class EOTestSuite
extends junit.framework.TestSuite

TestSuite to facilitate testing of EOEnterpriseObjects. It handles forcing models to connect with a common connection dictionary and adaptor type and also the installation of GVCValidation. It only handles tests in which all the EOModels use the same connection dictionary.

databaseType

public String databaseType()
Returns the database type (e.g. DB2, Oracle, FrontBase, OpenBase, mySQL etc.) that is to be used for this test run. This should match the database name as used in the EOPrototypes prototypes entity for this database and also the the name used in testConfigurationDictionary()if that is relevant. Remeber - case counts! This class gets the database type from the DATABASE_TYPE_KEY in testConfigurationDictionary().

Returns:
the database type that is to be used for this test run

testConfigurationDictionary

protected com.webobjects.foundation.NSDictionary testConfigurationDictionary()
Returns the configuration dictionary for this test run. This is used to implement the other methods in this class. This method can be overridden to get the dictionary from some other source, or the other methods can be re-implemented in a different manner. By default, this class gets the dictionary from the TestConfigurationDictionary.plist of the framework in the working directory when the tests are run.

The dictionary is expected to have this format: { databaseToUse = ""; connectionDictionaries = { = { URL = ... etc }; = { URL = ... etc }; }; modelsToIgnore = (); };

Returns:
the configuration dictionary for this test run
