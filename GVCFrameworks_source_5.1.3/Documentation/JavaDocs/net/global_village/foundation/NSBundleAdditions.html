findTableWithName

protected static com.webobjects.foundation.NSDictionary findTableWithName(com.webobjects.foundation.NSBundle bundle,
                                                                          String tableName)
Deprecated. not functional as of WO 5.4, default resources now in Default.lproj for WOResourceManager and NSBundle can't find them

Returns the localized string table tableName, or null if the file could not be found.

This does not use Collection.collectionWithContentsOfFile() because the path returned by resourcePathForLocalizedResourceNamed() is not fully qualified (and there is no way to get the FQ name!).

Note: uses the server's locale for localization. This is not appropriate for web apps.

Parameters:
bundle - the bundle in which to look for tableName.
tableName - the string table for which to search.
Returns:
a string table or null.
