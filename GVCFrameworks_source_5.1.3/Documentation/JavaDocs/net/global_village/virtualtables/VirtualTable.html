Makes sure that all the rows and fields are in memory so that we can do in-memory evaluation without firing off a million faults. We make use of the pre-fetching facilities of fetch specs to accomplish this. Note that this is likely not really needed due to the batch faulting facility of the EOModel.

refreshTableData

public void refreshTableData()
This method should be called after direct SQL manipulation of the data in this table (i.e. importing data or deleting rows). The data will have been modified in the database, but EOF doesn't know about that. Make EOF refault the table and its columns so that the relationships referring to these directly modifed objects get reloaded.

                        String delimiter,
                        boolean skipFirstLine,

                 throws IOException,
                        ParseException
Appends or replaces rows to the table from the given stream. The stream must contain character data in delimiter delimited format. Note: right now, this method doesn't handle quoted data or anything fancy. Also note that this method requires that the table has no changes on it... call saveChanges() before using this.

Overrides:
importTable in class Table
Parameters:
stream - the stream of characters that will make up the new rows. The stream is not closed at the end of this method - you'll have to do it yourself
sender - an arbitrary object that is used as a parameter to Column's valueForImportedValue() method
columnsToImport - an ordered list of columns that we will import. The first item in this array cooresponds to the first field in the data file, the second item to the second field, and so on
formatters - mapping of column names to java.text.Formats, used to format the field strings into objects of the correct type. You'll normally only need this for date/times and possibly for custom column types
delimiter - the character to use as the delimiter
skipFirstLine - should we skip the first line? Useful if, for example, the first line contains the column names
shouldReplace - true if this should replace all the rows, false if it should append the rows
Throws:
IOException - occurs when an IO exception occurs on the steam
ParseException - occurs when a formatter cannot format the data given it
