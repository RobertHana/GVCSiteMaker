Introduction

This document describes how to configure and install the GVC.SiteMaker Open Source distribution

Note: this installation process is designed for use on OS X and other UNIX systems. You will have some manual work to do on Windows.

Requirements

GVC.SiteMaker has the following requirements:

Java 1.5.0 Runtime

GVC.SiteMaker will not run with earlier versions and has not been tested against later versions.

Licensed version of WebObjects 5.4.3

A deployment installation of WebObjects will work for Ant builds (and for Eclipse builds, but you won't have any of the other development tools such as WOBuilder and EOModeler). You can get a licensed version of WebObjects from these sources
  • Apple Store
  • Purchase of an X Serve
  • ADC, register for developers evaluation copy for OS X
  • Request developers evaluation copy for Windows

SQL Database with JDBC connectivity

GVC.SiteMaker has been most often tested with FrontBase and only SQL scripts for that database are provided. EOPrototypes are also provided for Oracle and MySQL. These may not be fully up to date and / or tested. If not using FrontBase, you will need to generate your own SQL scripts to use in place of the FrontBase ones. Other databases can be supported by the addition of a database specific set of EOPrototypes to the GVCEOPrototypes project. Contact GVC at info@global-village.net for help porting GVC.SiteMaker to a different database.

GVC Frameworks

GVC.SiteMaker makes extensive use of the GVC Frameworks. These frameworks are included in this distribution bundle.

Contents

The Open Source distribution is arranged into these directories:

Install/Archive/ - the packages to be installed:
  • GVCSiteMakerFrameworks.tar.gz - the Open Source GVC Frameworks and GVCSMCore
  • GVCSiteMakerCustomFrameworks.tar.gz - usually contains only GVCSMCustom framework for a specific installation
  • GVCKerberos.tar.gz - this framework is needed only when Kerberos authentication is in use
  • GVCSiteMakerApplications.tar.gz - the GVCSiteMaker, GVCSMAdmin, and GVCSMUtility applications
  • GVCSiteMakerDocumentRoot.tar.gz - the core document root resources to be installed for the web server
  • GVCSiteMakerCustomDocumentRoot.tar.gz - document root resources unique to this installation to be installed for the web server
  • GVCSiteMakerMisc.tar.gz - contains seldom needed functionality to repair data and assist in data conversions between versions
Install/Script/ - Installation script and resources

Installation

As you are reading this, it is assumed that you have unpacked the installation bundle. You may now need to move it. See the next section.

Install the Bundles

We use an installation script to automate the install. First, you should do is to read the instructions on how this installation process works. The installation script is Install/Script/WOInstall.sh.

To make a fresh installation from scratch without backups: 

cd Install/Script
sudo ./WOInstall.sh <Installation Directory>
e.g. sudo ./WOInstall.sh /Users/sitemaker

Configure GVCSMCustom Framework

The GVCSMCustom framework contains configuration settings, and optionally code and components, that are unique to each installation. Unless you are installing a previously customized version, you are going to need to setup the configuration. You will find this framework in ~/Frameworks. Instructions are provided on configuration options.

Follow the Installation Guide

The Deployment Instructions will guide you through the rest of the installation process. By following the instructions here, you will have already performed some of the steps in the Deployment Instructions.
