net.global_village.woextensions.components
Class DataGroup

java.lang.Object
  net.global_village.woextensions.components.DataGroup

public class DataGroup
extends Object

A DataGroup organizes a sorted group of similar Enterprise Objects and is able to split it into sub-subgroups to model a hierarchy of EOs. It is intended to be used as the model for drill down components. Representative or aggregate values from the contained EOs of the DataGroup are accessed through Key-Value coding. Note that a DataGroup object only handles one level of hierarchy (other level are handled by other DataGroups). To create a DataGroup, you need to pass in the following:

  • data - an array of EOs to be managed by this DataGroup; all the EOs must be of the same type family, which means that all these EOs must respond to the same keypaths and compare methods passed during its creation)
  • groupAttributeKeyPath - the attribute keyPath used for grouping this data; if you want to perform calculations for a specific attribute, use this format "@.attribute", ie "@sum.hours", "@avg.hours"
  • attributeKeyPaths - the ordered array of attribute keyPaths that will determine the groupAttributeKeyPath to be used to generate its subGroups; groupAttributeKeyPath of this DataGroup must also belong to attributeKeyPaths;
  • comparators - the ordered array of NSComparator instances that will be used when generating the subGroups; it is expected that the NSComparator at a particular index will result in objects which are ordered in a sensible way with respect to the attributeKeyPath at the matching index. (WHAT DOES THIS MEAN? - ch: If they do not match, more than one group with the same title will be created.)
Below is an example that creates a DataGroup for the array of EOCustomObjects, the arrayOfWorkDetails(). It is created for the attribute key path "project.name" which is the first level of the hiearchy. From these param values, this DataGroup's subGroup will contain dataGroups for "taskArea.area.name", which is the next attribute key path from the attributeKeyPaths array.:
DataGroup newDataGroup = new DataGroup("project.name", arrayOfWorkDetails(), new Array(new Object[] {"project.name", "taskArea.area.name", "taskArea.task.name", "user.fullName", "@sum.hours"}), new Array(new Object[] {"compareByProject", "compareByArea", "compareByTask", "compareByUser", "compareByHours"})), Note that a DataGroup does not require valid attributeKeyPaths for it to function properly, you can pass in a dummy attribute key path like "$dummyAttribute" to initialize your DataGroups. For example, if you want to create the root DataGroup for an array of workDetails with the "project.name" as the first level, you can create it using this... DataGroup newDataGroup = new DataGroup("$dummyAttribute%", arrayOfWorkDetails(), new Array(new Object[] {"$dummyAttribute%", "project.name", "taskArea.area.name", "taskArea.task.name", "user.fullName", "@sum.hours"}), new Array(new Object[] {"$dummyCompare%", "compareByProject", "compareByArea", "compareByTask", "compareByUser", "compareByHours"})),
