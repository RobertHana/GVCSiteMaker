public class EOFValidationException
extends com.webobjects.foundation.NSValidation.ValidationException

An exception that contains a message key path, which can be used to retrieve a localized text message for display to the user.

Example validateForSave Usage:

 public void validateForSave() throws ValidationException
 {
    NSMutableArray exceptions = new NSMutableArray();

    try
    {
        super.validateForSave();
    }
    catch (NSValidation.ValidationException e)
    {
        exceptions.addObject(e);
    }

    if (numPages().intValue() > ( marriage().pagesPerSheet().intValue() * numSheets().intValue()) )
    {
        exceptions.addObject(new EOFValidationException(this, "numPagesNumSheetsMismatch");
    }

    if (exceptions.count() > 0)
    {
        throw NSValidation.ValidationException.aggregateExceptionWithExceptions(exceptions);
    }
 }

Example validateProperty Usage:

 public Object validateNumPages(Object value)
 {
     if ( numPages().intValue() < 2 || numPages().intValue() % 2 != 0 )
     {
         throw new EOFValidationException(this, NUMPAGES, EOFValidation.InvalidValue, value);
         // OR throw EOFValidationException.invalidValue(this, NUMPAGES, value);
     }
     return value;
 }

This is the constructor to use for validations that affect more than one property, for example in validateForSave(). partialPath must be a valid failure message key path in the form of <property key>.<failure key> or just <failure key>. The entity name is extracted from the eo and this partial path is appended to it to form a full messageKeyPath.

EOFValidationException

public EOFValidationException(Object object,
                              String className,
                              String propertyName,
                              String failureKey,
                              Object aFailedValue)
Designated constructor. This is intended for internal use, but may be used for special purposes.

The message keypath will be formed as <className>.<propertyName>.<failureKey> or <className>.<failureKey> if propertyName is null.

Parameters:
object - optional Object that validation failed on, passed to NSValidation.ValidationException constructor as object
className - the "name" of object class that validation failed on, usually the EOF entity name but must match the name of a .strings file in the Resources.
propertyName - optional name of the attribute or relationship that failed to validate, passed to NSValidation.ValidationException constructor as key
failureKey - the key (usually an EOFValidation constant) that indicates the type of validation failure
aFailedValue - the value that failed validation
