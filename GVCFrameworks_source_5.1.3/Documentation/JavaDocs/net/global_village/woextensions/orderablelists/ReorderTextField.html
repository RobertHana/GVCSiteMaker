setSelectedIndex

public void setSelectedIndex(Integer newSelectedIndex)
Moves the related object to the selected index if this is not the current position. This component should not be used in a form with other inputs as it will change the list in the repetition while the takeValues phase is running. If this is to be used in a form with other inputs, some arrangement will have to be made with the container to delay reordering.

Parameters:
newSelectedIndex - the one based index to move this object to

Overridden so that the new order set during the takeValues phase can be processed before the form's action runs. This is a hack and may cause future bugs. Be suspicious of this method! It was done this way to see if we could find a solution where the value is taken without altering the collection being iterated during takeValues.
