public class StrictDateFormatter
extends com.webobjects.foundation.NSTimestampFormatter

A simple sub-class (replacement) of NSTimestampFormatter that works around bugs in NSTimestampFormatter and makes parsing of date strings strict (see java.text.DateFormat method lenient()). NSTimestampFormatter is lenient in that it accepts months > 12, days > 31 etc. As long as the format somewhat matches the entered text it will produce some sort of date. This is often not desireable for input validation. This class will throw a ParseException if the parsed text does not strictly match the pattern and if any values are not in a normal range. This ParseException will then result in public void validationFailedWithException(Throwable t, Object value, String keyPath) being called if this formatter is used on a WOForm input component.

This is implemented using SimpleDateFormat so only the Java format patterns from java.text.SimpleDateFormat are supported.
