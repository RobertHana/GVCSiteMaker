Dangling SiteFile References

Due to the problem described in SCR 833 it is possible for a PageComponent (Image, ListLink) to retain a reference
to a deleted SiteFile. This results in the Section (or link) using this PageComponent being unviewable and uneditable.
Below is a list of the currently affected PageComponents and the related Section and Website. These are fixed by
setting the file reference to null. You must fix these before fixing the orphaned PageComponents.
