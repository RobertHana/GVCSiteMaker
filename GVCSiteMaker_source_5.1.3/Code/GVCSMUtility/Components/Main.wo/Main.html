GVC.SiteMaker Utility

Conversion and Maintenance Functions

The code behind these functions should be regarded as experimental and approriate backups made before attempting any of these!

 1. A nasty little page to collect hacks to clean up SiteFiles from damage from stupidity and bugs in previous versions. There is also a function to recalculate actual quota usage to accomodate errors caused by previous versions. Ê

 2. There is no functionality in GVCSMAdmin to delete an OrgUnit that still has Websites and other OrgUnits under it. This page provides that functionality. Use with caution!
Force OrgUnit Deletion

 3. Due to a bug in GVCSiteMaker PageComponents can retain references to deleted SiteFiles or can become Orphaned when their Section is deleted.
Clean up Damaged Page Components

 4. Locates any pieces of deleted Websites (Sections and uploaded files at present) and removes them. This was done to accomodate Websites deleted with ealier versions which had defects in the deletion process.
Remove Website Pieces

 5. This was added to handle the need to perform bootstrap like loading of style templates after the initial loading. It does not create any new styles, but does replace the templates for the existing styles.
Reload all Styles

 6. Load LDAP Server information. This allows LDAP access groups to be created..
Load LDAP Servers

 7. Edit the bindings of a PageComponent (AKA SM_PAGE_COMP). Short, brutish, and nasty.

8. Send a message to internal users, external users, or all users
Send Message to Users

9. Remove duplicate columns from a data access section.
Remove Duplicate Columns
